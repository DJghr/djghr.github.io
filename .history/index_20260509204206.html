Veille Technologique

Une démarche proactive pour anticiper les risques, sécuriser mes développements et comprendre les enjeux de demain.

Ma problématique

« Comment l'IA transforme-t-elle la cybersécurité, entre automatisation des attaques et protection des données sensibles (santé et entreprise) ? »

Bien que ma veille technologique ne soit pas toujours le point de départ de mes lignes de code, elle est le pilier de mon audit de sécurité. Elle me permet de synthétiser mes recherches sur les risques liés à mes réalisations et de garantir l'intégrité de mes applications.

Démarche et Sources

Afin de rester continuellement informé sur les vulnérabilités actuelles, j'utilise Feedly pour centraliser et catégoriser mes flux d'informations. Mes sources de référence incluent :

  • Le CERT-FR : pour les alertes et réponses aux attaques.
  • Zataz : pour suivre l'actualité des cybermenaces.
  • La CNIL : pour les normes de protection des données (RGPD).

Le rôle de l'IA

Je développe avec l'assistant Gemini intégré dans VS Code pour gagner en efficacité algorithmique. Cependant, ma veille me démontre régulièrement que le code généré automatiquement comporte des risques.

Cette sensibilisation m'impose une rigueur d'audit : je relis et sécurise systématiquement le code proposé par l'IA pour éviter l'introduction de failles invisibles.

Application concrète à mes projets

Projet GSB

L'actualité récurrente des fuites de données (identifiées via Zataz) m'a fait comprendre l'importance critique de la sécurisation des accès métiers.

Dans l'application GSB, j'ai donc implémenté le hachage en SHA-256 pour les mots de passe et renforcé la protection grâce à l'authentification Basic Auth.

Projet Cas Liémie (Android)

Le traitement des tournées d'infirmières implique des données de santé, exigeant une confidentialité stricte dictée par la CNIL.

Ma veille m'a alerté sur les vulnérabilités du stockage local (DB4O) sur mobile. J'ai donc été particulièrement vigilant sur la sécurisation des flux JSON échangés avec l'API REST.
