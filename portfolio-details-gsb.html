Projet : Application de Gestion des Rapports de Visite (GSB)

1. Présentation du Contexte

Le laboratoire Galaxy Swiss Bourdin (GSB) souhaite moderniser ses outils numériques pour faciliter le travail de ses visiteurs médicaux. L'objectif principal de ma mission était d'informatiser les comptes-rendus de visites pour mieux définir les objectifs commerciaux et l'attribution des budgets semestriels.

2. Ma Mission : La Gestion des Rapports de Visite

En tant que développeur responsable de la Mission 2, j'ai conçu et réalisé un module permettant aux visiteurs médicaux de :

  • S'authentifier de manière sécurisée pour accéder à l'application.
  • Saisir et modifier des rapports détaillés (choix du médecin, motif de la visite, bilan de l'entretien).
  • Gérer les échantillons : enregistrer les médicaments présentés et les quantités offertes lors de chaque visite.
  • Consulter l'historique : afficher et modifier les anciens rapports pour un médecin spécifique.

3. Environnement Technique & Méthodologie

Pour ce projet, nous avons travaillé dans un contexte professionnel strict :

  • Langages & Outils : Développement en programmation orientée objet avec Visual Studio 2022 et le framework .NET.
  • Base de Données : Implémentation sur un serveur SQL distant en respectant un schéma relationnel mis à jour pour les nouveaux besoins.
  • Gestion de Version : Utilisation de GitHub pour le versionnage du code et le travail collaboratif.
  • Méthodologie Agile (SCRUM) :
    • Planification via des Sprints et découpage en User Stories.
    • Mêlées quotidiennes de 15 minutes pour synchroniser l'avancement (tâches réalisées, difficultés rencontrées, objectifs du jour).
    • Suivi des tâches via l'outil collaboratif Trello.

4. Mes Réalisations (Livrables)

  • Dossier Fonctionnel : Synthèse du contexte, nouveaux besoins et schéma de la base de données actualisé.
  • Interface Homme-Machine (IHM) : Création de maquettes respectant la charte graphique de GSB et les normes de qualité.
  • Jeux d'essais : Création de données de test pour valider le bon fonctionnement de chaque module.
  • Documentation Technique : Présentation de l'enchaînement des formulaires et de l'architecture client-serveur.
